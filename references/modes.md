# Modes

## print-formal

Use for document-style Markdown that should feel like a finished paper document.

Best for:
- Chinese long-form writing
- reports, proposals, plans, articles
- print-first output
- stable heading hierarchy and page layout

Default assumptions:
- A4
- LXGW WenKai body text
- body size must be validated against actual exported PDF output, not just template settings
- academic-inspired tone without strict journal formatting

## web-clean

Use when the source is structurally closer to a webpage or README.

Best for:
- GitHub README export
- Markdown with lots of web-oriented rhythm
- style-heavy technical docs
- fallback when formal rendering is not the best visual fit
